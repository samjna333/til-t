### Context
n8n을 사용하여 디스코드에서 메시지를 수신하면 자동으로 워크플로우가 트리거되도록 설정하였습니다. 이 워크플로우는 메시지를 TIL 형식으로 변환하여 GitHub에 푸시하고, README.md 파일에 업데이트를 추가한 후 완료 메시지를 발신합니다. 

### Core
```javascript
{
  "nodes": [
    {
      "parameters": {
        "webhookMethod": "POST",
        "path": "discord-webhook",
        "responseCode": 200
      },
      "name": "Discord Trigger",
      "type": "n8n-nodes-base.webhook",
      "typeVersion": 1
    },
    {
      "parameters": {
        "repository": "리포지토리 이름",
        "branch": "main",
        "filePath": "경로/TIL.md",
        "fileContent": "{{ $node[\"Discord Trigger\"].json[\"body\"] }}"
      },
      "name": "GitHub Push",
      "type": "n8n-nodes-base.github",
      "typeVersion": 1
    },
    {
      "parameters": {
        "channel": "#notifications",
        "text": "워크플로우가 성공적으로 완료되었습니다."
      },
      "name": "Send Discord Message",
      "type": "n8n-nodes-base.discord",
      "typeVersion": 1
    }
  ]
}
```

### Insight
이 워크플로우는 디스코드에서 메시지를 수신하면 자동으로 GitHub에 TIL 형식으로 기록하고, 이후 완료 메시지를 재발신하여 프로세스의 완료를 통보합니다. n8n과 GitHub 간의 원활한 연동을 통해 자동화를 구현할 수 있어 효율적입니다.
